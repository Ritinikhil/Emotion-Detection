## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion for improvement:

1. Check if the issue already exists in the [Issues](https://github.com/Ritinikhil/Emotion-Detection/issues) tab
2. If not, create a new issue with a clear title and description
3. Include steps to reproduce (for bugs) or detailed explanations (for features)

### Pull Requests

We welcome pull requests! Here's how to contribute:

1. **Fork the repository** to your GitHub account
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/Emotion-Detection.git
   cd Emotion-Detection
   ```
3. **Create a new branch** for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** and test thoroughly
5. **Commit your changes** with clear commit messages:
   ```bash
   git commit -m "Add feature: description of your changes"
   ```
6. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Open a Pull Request** from your fork to our main branch

### Development Guidelines

- Follow PEP 8 style guidelines for Python code
- Add comments for complex logic
- Update documentation if you change functionality
- Test your changes with both FER2013 and RAF-DB datasets when applicable
- Ensure your code doesn't break existing functionality
